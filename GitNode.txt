		config --global user.name ""
		config --global user.email ""
		add
		commit
		status
		log
		log --pretty=oneline
		log --oneline
		reflog
		reset --hard 索引——>:git reflog命令产生的索引，使用hard参数回退版本会改变本地库、暂存区、工作区
		reset --mixed 索引,mixed参数 会改变本地库和暂存区
		reset --soft 索引，soft参数 仅改变本地库


		---多分支的操作---
		branch -v   		--> 查看分支 带*的就是当前分支
		branch 分支名  		-->创建一个新分支，会自动把master主分支内最新的版本，copy到新分支 
		checkout 分支名 		-->切换分支
		merge 分支名 （切换到master主分支后再进行该命令） -->合并分支 2个分支中 对同一文件进行了不同更改的话 会有合并冲突 Git不会删掉某一分支的更改 都留着，人为决定 留哪个。更改完之后文件处于工作区，需要放进暂存区、本地库，在执行git commit的时候，不要加文件名。--> 示例：git commmit -m "解决了合并冲突"



		---本地库与远程库的操作---
		remote -v                -->查看远程库的别名 执行无效就说明 本地库和远程库并无联系交互
		remote add 别名 远程库地址  -->为特定远程库地址 起一个别名
		push 远程地址\别名 本地分支名     -->将本地分支的版本信息内容 推送到远程库
